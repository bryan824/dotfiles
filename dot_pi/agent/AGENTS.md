# Agent guidance

For any multi-step code or product change, route through the workflow skills
instead of jumping straight to edits: pick the skill by the situation, honor the
seams — an approved `design` before `implement`, and `verify` before `commit` —
and consult the workflow set's `RESOLVER.md` for the full routing table and the
modes to enter at (`debug`, `survey`, `prototype`, `commit`, …).
